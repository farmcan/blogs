# SEO Guide

Optimize for search without making the prose mechanical.

## Title

- Put the main topic in the title naturally.
- Keep it specific. "How We Reduced Build Times by 40 Percent" is stronger than "Performance Improvements".
- Avoid vague clickbait.

## Summary

- Treat `summary` as the meta description baseline.
- Aim for one sentence that states the topic, audience, and value.
- Write it for humans first.

## Headings

- Use headings that reflect actual reader questions or decision points.
- Keep one concept per heading.
- Do not repeat the title verbatim in every subheading.

## Keywords

- Pick one primary phrase and a few natural supporting terms.
- Use the primary phrase in the title, intro, and one relevant heading if it fits naturally.
- Do not force the keyword into every paragraph.

## Links

- Use descriptive anchor text.
- Link to supporting docs, release notes, or prior posts when helpful.
- Avoid "click here" unless context already makes the target obvious.

## Snippets And Search Results

- Front-load the first paragraph with concrete context. Search previews often pull from the opening lines.
- If the post includes steps, comparisons, or definitions, keep those sections crisp so they can be excerpted cleanly.

## Common SEO Failures

- Title is catchy but vague.
- Intro takes too long to name the topic.
- Headings are generic.
- Keyword usage sounds unnatural.
- The post promises one thing and delivers another.
